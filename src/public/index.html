Kubernetes
Deployment Strategy

Rolling update with zero downtime — no pod is terminated until its replacement is healthy.

Zero-Downtime Rolling Update

strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1 # +1 new pod added first
    maxUnavailable: 0 # never reduce below desired
minReadySeconds: 10 # wait before continuing
replicas: 3 # default replica count

Security
Defense in Depth

Every layer secured — from container runtime to network policies to secret management.

✓

Non-Root Container

Runs as UID 1001. Read-only filesystem. All Linux capabilities dropped.

✓

RBAC

Least-privilege ServiceAccount. Pod Security Standards — restricted profile enforced.

✓

Trivy Image Scanning

Automated CVE scanning in CI. CRITICAL/HIGH vulnerabilities block the deployment.

✓

NetworkPolicy

Restricts pod-to-pod communication. Only required traffic is allowed.

✓

HashiCorp Vault

External Secrets Operator syncs secrets from Vault. No plaintext secrets in Git.

✓

ResourceQuota

Namespace-level CPU, memory, and object count limits enforced.

✓

CodeQL Scanning

Automated static code analysis via GitHub CodeQL v4 with SARIF reporting.

✓

Rate Limiting

NGINX Ingress enforces per-IP rate limits. Protects against DDoS and abuse.

Quick Start
Get Running in Minutes

Three ways to run the stack — local dev, Docker Compose full stack, or full Kubernetes.

1

Local Development

# Install dependencies
npm install
# Run tests
npm test
# Start dev server (auto-reload)
npm run dev
# → http://localhost:3000
2

Docker Compose Stack

# Full stack: app + Prometheus + Grafana + Loki
docker compose up -d --build
App: http://localhost:3000
Prometheus: http://localhost:9090
Grafana: http://localhost:3001
(admin/admin)
3

Kubernetes (Minikube)

# Start Minikube
minikube start
# Build image inside Minikube
minikube docker-env | Invoke-Expression
docker build -t devops-demo:latest .
# Deploy with Helm
helm upgrade --install devops-demo helm/devops-demo \
-f helm/devops-demo/values-dev.yaml \
--namespace devops-demo-dev --create-namespace
# Access the app
kubectl port-forward svc/devops-demo 8080:80
